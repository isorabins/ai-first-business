# AI File Organization System

*How to build an AI that knows where all your files are — before you ask*

---

## The Problem

You have files everywhere. Multiple Google Drive accounts, maybe a Dropbox, maybe some repos. You've lost track of half of it. When you need something, you either search and hope, or you just know it's gone.

The real cost isn't the time you spend searching. It's that your AI assistant is just as lost as you are. You ask "find that Forage Bali partnership agreement" and it has to guess, or ask you, or give up.

This system fixes that.

---

## What It Does

Runs in the background and builds three things:

1. **Drive indexes** — a Markdown file per Google account listing every file (name, type, size, folder path)
2. **AI summaries** — a 1-2 sentence description of what's actually *in* each file, generated by vision/text AI
3. **A master map** (`DRIVE-MAP.md`) — the single entry point that points to everything

Once built, your AI can find any file in 3 hops: map → account index → file summary. No searching. No guessing.

We did this across 5 Google Drive accounts (~37,000 files, ~188 GB). The AI now knows what's in all of them.

---

## What You'll Need

- **OpenClaw** running (the AI agent platform this is built on)
- **Python 3.10+**
- **Google OAuth tokens** for each Drive account (one-time setup per account)
- **ffmpeg** (for video frame extraction — `apt install ffmpeg` or `brew install ffmpeg`)
- **pdftotext** (for PDFs — `apt install poppler-utils` or `brew install poppler`)
- **An AI model** with vision support via OpenClaw (Claude, GPT-4o, or Gemini all work)

**API costs:** Running vision on ~20,000 files costs roughly $5–15 in API credits using GPT-4o-mini or Gemini Flash. It runs over days/weeks so there's no single big bill — just small charges per batch.

## The Core Script: `file-summarizer.py`

The main script is `scripts/file-summarizer.py` in this repo. Here's what it does:

- Reads your Drive index files to know what exists
- Downloads each file temporarily
- For images/video frames: sends to vision AI for a description
- For docs/PDFs/spreadsheets: extracts text, sends to AI for a summary
- Writes the summary back to a companion index file
- Tracks state in a JSON checkpoint so it resumes exactly where it left off if interrupted

### How it handles different file types

| File type | Approach |
|-----------|----------|
| Images (JPEG, PNG, HEIC) | Convert to JPEG → vision model → 1-sentence description |
| Videos (MP4, MOV, etc.) | Extract frame at 1s → vision model → 1-sentence description |
| Google Docs / Slides / Sheets | Export as plain text → text model → 2-3 sentence summary |
| PDFs | Download → pdftotext (first 5 pages) → text model → summary |
| DOCX | Download → python-docx → text model → summary |
| Audio / .m4b audiobooks | Stub: `(audio: filename)` — can't summarize audio with vision |
| Bulk AI image sequences | Stub: `(image: filename)` — not worth spending API credits |

Expect ~83% real summaries, ~17% graceful stubs. The stubs are correct behavior, not failures.

## Setup

### Step 1: Get Google OAuth tokens

For each Drive account you want to index, you need an OAuth token with `drive.readonly` scope.

The easiest path: if you're already using OpenClaw with Gmail connected for any account, you likely already have the token. Check `/root/.credentials/email-tokens/` — you'll see JSON files named like `iso_at_foragesf_com.json`. Each one gives Drive access to that account.

If you need to create tokens from scratch, the OpenClaw docs cover Gmail OAuth setup — Drive uses the same token with the same scopes.

Token format expected by the script:
```json
{
  "token": "ya29.xxx",
  "refresh_token": "1//xxx",
  "token_uri": "https://oauth2.googleapis.com/token",
  "client_id": "xxx.apps.googleusercontent.com",
  "client_secret": "xxx"
}
```

### Step 2: Build the initial drive index

Before the summarizer can run, you need an index of what files exist. This is a separate step that walks your Drive and writes a Markdown file listing every file.

We built a custom indexer for this — it's not in the public repo yet (coming soon), but the format is simple enough to build yourself or adapt from any Drive listing script:

```markdown
# ForageSF Drive Index
*Last updated: 2026-03-23 | 14,500 files*

## Folder: CSA - Forage Box/forageSF Media/
- `class-photo-2019-04.jpg` | image/jpeg | 3.2 MB | 2019-04-15
- `underground-market-flyer.pdf` | application/pdf | 0.8 MB | 2018-06-02
...
```

One file per Drive account, saved to `memory/drive-indexes/<account>.md`.

### Step 3: Configure the summarizer

Edit the top of `file-summarizer.py` to point to your setup:

```python
TOKEN_DIR = Path("/path/to/your/tokens")
INDEX_DIR = Path("/path/to/your/drive-indexes")
STATE_FILE = Path("/path/to/state.json")

ACCOUNTS = {
    "my-account": {
        "token": "my_account_token.json",
        "email": "me@example.com",
        "label": "My Account",
    },
    # add more accounts...
}

# Your AI endpoint (OpenClaw gateway, or any OpenAI-compatible API)
GATEWAY_URL = "http://127.0.0.1:18789/v1/chat/completions"
GATEWAY_TOKEN = "your-token-here"
GATEWAY_MODEL = "gpt-4o-mini"  # or claude-3-haiku, gemini-flash, etc.
```

### Step 4: Initialize and run

```bash
# Initialize state (creates the state file)
python3 scripts/file-summarizer.py --init

# Run a test batch (10 files, dry run to see output without writing)
python3 scripts/file-summarizer.py --batch-size 10 --dry-run

# Start for real (processes 20 files per run)
python3 scripts/file-summarizer.py --batch-size 20

# Check progress at any time
python3 scripts/file-summarizer.py --status
```

### Step 5: Set up a cron job to run it continuously

The summarizer is designed to run as a repeating background job. In OpenClaw, you'd add a cron that runs every 30 minutes and calls the script. You can also just run it in a loop or a screen session:

```bash
# Simple loop approach
while true; do
    python3 scripts/file-summarizer.py --batch-size 20
    sleep 1800  # 30 minutes
done
```

At 20 files per 30-minute batch, it processes ~960 files per day. For 20,000 files, expect ~3 weeks to complete at this rate — or run larger batches if you want it done faster.

### Step 6: Build DRIVE-MAP.md

Once you have indexes for your accounts, write a master map file that describes each account and points to its index. This is what your AI reads first when you ask it to find something.

```markdown
# DRIVE-MAP — Where Everything Lives

## me@example.com — Main Account
~5,000 files | 25 GB
Work files, projects, invoices.
Overlay index: memory/drive-indexes/main.md

## personal@gmail.com — Personal
~3,000 files | 18 GB
Photos, health records, travel docs.
Overlay index: memory/drive-indexes/personal.md
```

Add this to your AI's workspace so it gets loaded into context. Now your AI knows where to look before it searches.

## How Your AI Uses It

With the map and indexes in place, your AI can:

**Find a specific file:**
> "Find the Forage Bali waiver"
> → Reads DRIVE-MAP.md → sees forage-bali index → scans summaries → finds it in 3 hops

**Answer questions about file contents:**
> "What were the key terms in the partnership agreement?"
> → Finds file via summary → reads original if needed → answers

**Spot what's missing:**
> "Do I have a signed version of that contract?"
> → Scans summaries for relevant files → reports what it finds and what's absent

The AI doesn't need to download or search anything at query time — the groundwork is already done.

## FAQ

**Does it re-summarize files every time?**
No. State is tracked per file ID. Once a file has a summary, it's skipped. Only new files get processed on subsequent runs.

**What if it gets interrupted?**
The state file checkpoints after each file. Restart it and it picks up exactly where it stopped.

**Can I use this with Dropbox or OneDrive?**
The current script is Google Drive only. The architecture would work for other providers with a different download layer — the AI summarization part is provider-agnostic.

**What model should I use?**
For images and video frames: GPT-4o-mini or Gemini 1.5 Flash (fast, cheap, good enough).
For docs: same, or Claude Haiku.
Don't use expensive models for bulk summarization — you're not doing complex reasoning, just description.

**What about privacy?**
Files are sent to your AI provider's API. If you have sensitive files (medical records, legal docs), either skip those accounts or use a local/self-hosted model.
